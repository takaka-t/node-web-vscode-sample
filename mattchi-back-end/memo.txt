・セキュリティ
helmet 実装

helmetとは
CSP(bontents security policy)の設定ができ、HTTP ヘッダーを監視、管理、制御

helmetインストール
npm i helmet

型定義インストール
npm i -D @types/helmet

X-Powered-Byを非表示に
サーバーが何かをわからないようにする
helmetを使用すれば、デフォルトで設定される

CSP設定項目
公式に詳しく書いてある
https://helmetjs.github.io/
Content-Security-policy(XSS攻撃の軽減やデータインジェクション攻撃の軽減をするために追加できるセキュリティレイヤ)

cspチェッカー
https://csp-evaluator.withgoogle.com/


Cross-Origin-Opener-Policy: ページのプロセスを分離するのに役立ちます
Cross-Origin-Resource-Policy: 他のユーザーがあなたのリソースをクロスオリジンでロードすることをブロックします
Origin-Agent-Cluster: プロセス分離をオリジンベースに変更します
Referrer-Policy:Refererヘッダーを制御します
Strict-Transport-Security: ブラウザに HTTPS を優先するよう指示します。
X-Content-Type-Options: MIME スニッフィングを回避します
X-DNS-Prefetch-Control: DNS プリフェッチを制御します
X-Download-Options: ダウンロードを強制的に保存します (Internet Explorer のみ)
X-Frame-Options:クリックジャッキング攻撃を軽減するレガシーヘッダー
X-Permitted-Cross-Domain-Policies: Acrobat などの Adob​​e 製品のクロスドメイン動作を制御します
X-Powered-By: Web サーバーに関する情報。単純な攻撃に使用される可能性があるため削除されました
X-XSS-Protection: XSS 攻撃を軽減しようとするレガシーヘッダーですが、事態を悪化させるため、ヘルメットで無効にします
